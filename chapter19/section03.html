泛型

在前面学习集合时，我们都知道集合中是可以存放任意对象的，只要把对象存储集合后，那么这时他们都会被提升成Object类型。当我们在取出每一个对象，并且进行相应的操作，这时必须采用类型转换。

大家观察下面代码：

public class GenericDemo {
    public static void main(String[] args) {
        Collection coll = new ArrayList();
        coll.add("abc");
        coll.add("123");
        coll.add(5);//由于集合没有做任何限定，任何类型都可以给其中存放
        for(Object obj : coll) {
            int length = ((String) obj).length();
            System.out.println("字符串的长度是" + length);
        }
    }
}

程序在运行时发生了问题java.lang.ClassCastException.

原因：由于集合中可以存储任意数据类型，当在集合中存入了字符串和数字等多种类型数据，在取出数据进行格式转换时，就会导致类型转换异常 ClassCastException。

解决方案：Collection虽然可以存储各种对象，但实际上通常一个Collection只存储同一类型的对象。例如，要存数字就全部存数字，要存字符串就全部存字符串。因此在JDK5之后，新增了泛型(Generic)语法，让你在设计API时可以指定类或方法支持泛型，这样我们使用API的时候也变得更为简洁，并得到了编译时期的语法检查。

  • 泛型：可以在类或方法中预支地使用未知的类型。

tips:一般在创建对象时，将未知的类型确定具体的类型。当没有指定泛型时，默认类型为Object类型。

泛型的作用

  • 将运行时期的ClassCastException，转移到了编译时期变成了编译失败。
  • 避免了类型强转的麻烦。
Collection<String> collection = new ArrayList<String>();

collection.add("hello");
collection.add("good");
collection.add("hi");
//collection.add(10);  编译时就直接报错，提示类型不匹配
for (String str : collection) {  // 不要再做强制类型转换的操作，取出的直接就是字符串类型
        System.out.println(str.length());
    }
}

tips:泛型是数据类型的一部分，我们将类名与泛型合并一起看做数据类型。

使用泛型

我们在集合中会大量使用到泛型，这里来完整地学习泛型知识。\泛型，用来灵活地将数据类型应用到不同的类、方法、接口当中。将数据类型作为参数进行传递。

语法格式：

修饰符 class 类名<代表泛型的变量> {  }

例如，API中的ArrayList集合：

class ArrayList<E>{ 
    public boolean add(E e){ }

    public E get(int index){ }
       ....
}

使用泛型： 在创建对象的时候确定泛型

例如，ArrayList<String> list = new ArrayList<String>();

此时，变量E的值就是String类型,那么我们的类型就可以理解为：

class ArrayList<String>{ 
     public boolean add(String e){ }

     public String get(int index){  }
     ...
}

再例如，ArrayList<Integer> list = new ArrayList<Integer>();

此时，变量E的值就是Integer类型,那么我们的类型就可以理解为：

class ArrayList<Integer> { 
     public boolean add(Integer e) { }

     public Integer get(int index) {  }
     ...
}

自定义泛型类

在定义类的使用使用了泛型，那么这个类我们可以称之为泛型类。通过泛型可以完成对一组类的操作对外开放相同的接口，例如我们上面提到的Collection,ArrayList等类，在定义时都使用到了泛型，都可以被称为泛型类。

public class Test {
    public static void main(String[] args) {
        // 创建时指定了 T 的类型为 String
        MyGeneric<String> mg1 = new MyGeneric<String>("hello");
        System.out.println(mg1.getT());
        mg1.setT("good");
        System.out.println(mg1.getT());


        MyGeneric<Integer> mg2 = new MyGeneric<Integer>(3);
        System.out.println(mg2.getT());
    }
}

class MyGeneric<T> {  // T 并不是一个切实存在的类型,由创建对象时的代码指定

    // 声明类时，使用泛型语法，定义了一个泛型 T,后续可以直接使用泛型
    private T t;
    public MyGeneric(T t) {
        this.t = t;
    }
    public T getT() {
        return this.t;
    }

    public void setT(T t) {
        this.t = t;
    }
}

含有泛型的方法

泛型不仅可以定义在类里，同时还可以在方法中使用泛型。

修饰符 返回值类型<代表泛型的变量>或<代表泛型的变量> 方法名(参数){  }

public class Test {
    public static void main(String[] args) {
        // 创建对象
        MyGenericMethod mm = new MyGenericMethod();
        System.out.println(mm.demo("hello"));
    }
}

class MyGenericMethod<T> {  // 在声明类时定义了泛型 T

    // 在方法里又使用 <E> 声明了一个新的泛型类型 E
    public <E> E demo() {
        return null;
    }

    // 在方法里再使用 <F> 声明一个新的泛型
    public <F> void test(F f) {
        System.out.println("hello");
    }
}

含有泛型的接口

用法和泛型类基本一致。

public interface MyGenericInterface<E>{
    public abstract void add(E e);
    public abstract E getE();  
}

泛型通配符

当使用泛型类或者接口时，传递的数据中，泛型类型不确定，可以通过通配符<?>表示。但是一旦使用泛型的通配符后，只能使用Object类中的共性方法，集合中元素自身方法无法使用。

public class Test {
    public static void main(String[] args) {
        ArrayList<String> names = new ArrayList<String>();
        names.add("hello");
        names.add("good");
        names.add("hi");
        printElement(names);

        ArrayList<Integer> nums = new ArrayList<Integer>();
        nums.add(1);
        nums.add(2);
        nums.add(3);
        printElement(nums);
    }

    public static void printElement(Collection<?> coll) {
        for (Object obj : coll) {  // 这里的类型只能当做 Object,不能使用String或者Integer
            System.out.println(obj);
        }
    }
}

注意：泛型里不存在继承，ArrayList<Object> arr = new ArrayList<String>(); 这种语法是错误的。

思考：使用了泛型通配符以后，就无法再确定数据的类型了，那使用通配符的意义在那儿？

通常情况下，我们不会直接只使用泛型通配符，而是使用泛型通配符再结合super和extends来规定泛型的上下限。

泛型上限

  • 格式： 类型名称 <? extends 类 > 对象名称
  • 意义： 只能接收该类型及其子类

泛型下限

  • 格式： 类型名称 <? super 类 > 对象名称
  • 意义： 只能接收该类型及其父类
package com.atguigu.java;


import java.util.ArrayList;
import java.util.Collection;

public class Test {
    public static void main(String[] args) {
        Collection<Person> ps = new ArrayList<Person>();
        Collection<Student> ss = new ArrayList<Student>();
        Collection<Animal> as = new ArrayList<Animal>();
        Collection<Creature> cs = new ArrayList<Creature>();

        demo(ps);
        demo(ss);

        // 报错，demo方法需要的泛型是 Person类及其子类
        // demo(as);
        //demo(cs);

        foo(ps);
        foo(as);
        foo(cs);

        // 报错，foo 方法需要的泛型是 Person类及其父类
        //foo(ss);
    }

    public static void demo(Collection<? extends Person> people) {
        for (Person p : people) {
            p.work();
        }
    }
    public static void foo(Collection<? super Person> x) {

    }
}

class Creature {}

class Animal extends Creature {}

class Person extends Animal {
    private String name;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public Person(String name) {
        this.name = name;
    }

    public void work() {
        System.out.println(this.name + "正在工作");
    }
}

class Student extends Person {
    public Student(String name) {
        super(name);
    }
}
