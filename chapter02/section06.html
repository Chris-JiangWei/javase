入门程序HelloWorld

JDK安装完毕，可以开发我们第一个Java程序了。

Java程序开发三步骤：编写、编译、运行。

编写Java源程序

  1. 创建桌面上创建HelloWorld.txt，并将文件名修改为 HelloWorld.java.注意:文件的后缀名必须是.java!

  2. 用记事本或EditPlus等文本编辑器打开。

  3. 在文件中键入文本并保存，代码如下：

public class HelloWorld {
      public static void main(String[] args) {
        System.out.println("HelloWorld");
      }
}

友情提示：

每个字母和符号必须与示例代码一模一样，而且，所有的标点符号必须是英文的!

第一个HelloWord 源程序就编写完成了，但是这个文件是程序员编写的，JVM是看不懂的，也就不能运行，因此我们必须将编写好的Java源文件 编译成JVM可以看懂的字节码文件 ，也就是.class文件。

编译Java源文件

在DOS命令行中，使用cd命令进入到桌面，使用javac 命令进行编译。

javac HelloWorld.java

编译成功后，命令行没有任何提示。回到桌面，发现产生了一个新的文件 HelloWorld.class，该文件就是编译后的文件，是Java的可运行文件，称为字节码文件，有了字节码文件，就可以运行程序了。

Java源文件的编译工具javac.exe

运行Java程序

在DOS命令行中，进入.class文件所在的目录，使用java 命令进行运行。注意:在使用java命令运行代码时，不能添加.class后缀!!!

java HelloWorld

HelloWorld案例常见错误

  • 单词拼写问题(严格区分大小写)
    • 正确：class 错误：Class
    • 正确：String 错误：string
    • 正确：System 错误：system
    • 正确：main 错误：mian
  • 标点符号使用问题(所有的标点符号必须是英文标点)
    • 不能用中文符号，英文半角的标点符号(正确)
    • 括号问题，成对出现

注意事项

乱码

如果在代码里打印中文，执行代码时可能会出现中文乱码的问题。

乱码出现的原因：写源代码时的编码格式和编译代码时的格式不一致。

解决方案：将 源代码的编码格式和编译时的编码格式统一即可。

  • 在编辑器里修源代码的编码方式。
  • 在对源代码编译时，使用javac -encoding <编码方式> Test.java,指定编译时的编码方式。

类名和文件名的关系

类名和文件名可以不相同，但是，如果这个类名前面有 public修饰符，那么这个类的名字必须要和文件名相同。建议大家在定义类时，不管是否有public修饰符，都与源文件名保持一致，而且一个源文件尽量只写一个类，目的是为了好维护。

编码格式要求

  • 正确的注释和注释风格

    • 使用文档注释对整个文件、类或者方法进行注释。
    • 使用单行或者多行注释对方法里的代码进行注释。

  • 正确的缩进方式和空白

    • 使用一个tab键实现缩进
    • 操作符两边加空格，例如 int x = 3 + 2;

  • 推荐使用行尾风格的代码块。

    class Test
{   //  不推荐

}

class Test{  // 推荐

}
