方法的参数

方法的参数又被分为形式参数(形参)和实际参数(实参)。

  • 形参：声明方法时，在小括号内定义的数据，我们称之为形参。形参只是用来做占位，保证逻辑的完整性，仅从形参上不能知道函数的执行效果。
  • 实参：调用方法时，在小括号里传入的数据，我们称之为实参。实参是参与程序运行的具体值。
class Demo1 {
    public static void main(String[] args) {
        printSum(1, 2);  // 方法调用时出入的参数是实参，是程序运行时真正参与运算的值
    }

    // 方法声明时，定义的参数是形参
    public static void printSum(int a, int b) {
        int c = a + b;  // 形参只用来占位，保证逻辑的完整性
        System.out.println(c);
    }
}

可变参数

JavaSE 5.0 中提供了Varargs(variable number of arguments)机制，允许直接定义能和多个实参相匹配的形参。从而，可以用一种更简单的方式，来传递个数可变的实参。

语法:

// 声明方法
public static void test(int a, String... args){}

// 调用方法
test(1,"hello","good","hi","yes");

JDK5.0以前，可以使用数组表示可变参数:

// 形参里接收一个数组
public static void test(int a, String[] args) {}

// 调用时，实参传入一个数组
test(1, new String[]{"hello", "good"});

方法的返回值

使用关键字return来返回一个函数

  • 如果返回值类型不是void，方法体中必须保证一定有return 语句，并且要求该返回值的类型与声明的返回值类型一致或兼容。
  • 如果返回值类型为void时，return 后面不用跟返回值，甚至也可以没有return语句。
  • return语句表示方法的结束，后面就不能再写其他代码了，否则会报错：Unreachable code.
  • 一个方法可以有多个return语句，但是通常情况下，一个方法最多只会执行一个return语句。
  • 方法是否需要返回值，需要什么类型的返回值，由具体的业务逻辑决定。

方法的重载

  • 方法重载：指在同一个类中，允许存在一个以上的同名方法，只要它们的参数列表不同即可，与修饰符和返回值类型无关。
  • 参数列表：个数不同，数据类型不同，顺序不同。
  • 重载方法调用：JVM通过方法的参数列表，调用不同的方法。

练习

  1. 比较两个数据是否相等。参数类型分别为两个byte类型，两个short类型，两个int类型，两个long类型，并在main方法中进行测试。

  2. 判断一下哪些方法属于重载。

    public static void open(){}
public static void open(int a){}
public static void open(int a,int b){}            // 第3行和第8行报错，参数列表一样
public static void open(double a,int b){}
public static void open(int a,double b){}        // 第5行和第6行报错，参数列表一样
public static void open(int i,double d){}        // 第5行和第6行报错，参数列表一样
public static void OPEN(){}                        // 方法名不同
public static void open(int i,int j){}            // 第3行和第8行报错，参数列表一样
